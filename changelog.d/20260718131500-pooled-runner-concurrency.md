Add `backgroundJobs.pooledRunnerConcurrency` so each pooled child runs several jobs at once on its own event loop (total pooled capacity is `pooledRunnerCount × pooledRunnerConcurrency`), isolating a single job's failure to that job while a retiring child eagerly spawns a 1-for-1 replacement and only terminates once its in-flight set drains. Defaults to `1`, preserving the previous serial-per-child behavior.
