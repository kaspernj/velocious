# Changelog

- `SyncEnvelopeReplayService` now supports fully declarative configuration so apps override no hooks for the common shapes: token-model authentication (`authenticationTokenModel`/`authenticationTokenColumn`/`authenticationTokenParam`), a per-resourceType `applyHandlers` registry (functions or declarative field-map specs executed by the new `SyncReplayUpsertApplier`, which owns present-key filtering, typed coercion incl. sqlite booleans and dates, unknown-key rejection, upsert/delete, snapshot serialization, and domain `afterApply` tails), persist extension points (`persistExtraAttributes`, `persistSerializedData`), and declarative post-apply websocket fan-out (`broadcasts` + injected `broadcaster`). Unregistered resource types fail loudly. See `docs/sync-envelope-replay-service.md`.
