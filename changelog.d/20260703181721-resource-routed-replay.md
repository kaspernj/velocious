Resource-routed sync replay: `SyncEnvelopeReplayService` accepts `configuration`, `resourceTypeOverrides`, `ability`, `abilityContext` and `locals` and routes mutations without a matching applyHandler through the registered frontend-model resource classes — applying a mutation is just applying the permitted payload attributes to the model and saving it. Hooks on `FrontendModelBaseResource`: `authorizeSyncMutation`, `findSyncRecord` (ability-scoped), `applySync` (full escape hatch) and `afterSyncApply`, plus `syncAuthorizationFailureReason` for pinned denial reasons. Creates use the client-generated primary key with a save-then-check membership check; the envelope's `resourceId` stays authoritative over payload ids. Client-safe apply failures fail only their own sync (`{id, syncState: "failed", reason, message}`) and keep the batch going. `SyncResourceBase#buildReplayService` plumbs ability/context/configuration/locals under `replayServiceArgs()` and `replayServiceClass()` defaults to `SyncEnvelopeReplayService`. The `applyHandlers`/`SyncReplayUpsertApplier` path is deprecated (kept for released adopters, with precedence over routing).

`writableAttributes` moved up to `FrontendModelBaseResource` and became a plain permit list of camelCase attribute names (`static writableAttributes = ["title", "startsAt"]`), resolved through shared resources and used as the default `permittedParams`. The typed sync-attribute schema engine (`normalizeAttributesWithSchema`, typed entries, column-metadata inference, per-attribute message overrides) was removed: value casting belongs to the record layer (booleans, datetime strings and numbers cast on write) and validation belongs to model validations. BREAKING for pre-release schema adopters: object schemas and `normalizeWritableAttributes` are gone; pinned per-attribute validation message texts are replaced by the translated model-validation messages (reasons/codes stay stable).

Validation messages route through a translated message layer (`velocious.errors.messages.*` msgIDs via `configuration.getTranslator()`, Rails-style defaults): the presence/uniqueness validators, the new `length` validator (`validates("name", {length: {maximum: 255, minimum: 2}})` with translated too-long/too-short messages) and routed replay validation failures. Also adds `resolveFrontendModelResourceClass` for registry-based resource-type resolution honoring `modelName` overrides, and hardens the test runner: uncaught exceptions during a run become visible attributed failures (like #847 did for unhandled rejections) and `scripts/run-tests.js` prints the child's non-zero exit code instead of ending CI logs mid-run with no explanation.
