`Record.withAdvisoryLock` and `Record.withAdvisoryLockOrFail` now use driver-owned timeout cleanup when `holdTimeoutMs` fires, instead of trying to run a normal release query on that same potentially wedged session. Session-scoped drivers close the lock-owning database session, while SQLite releases its emulated lock state directly. This prevents MySQL/MariaDB named locks from staying held after `AdvisoryLockHoldTimeoutError` when the callback or connection has stopped making progress.
