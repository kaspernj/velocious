## Fixed

- The test runner now pins a database connection per test (around `beforeEach` + the test body + `afterEach`) instead of relying solely on a single suite-level pin. The suite-level pin could be orphaned mid-run when a spec closed the shared connections (e.g. tearing down a Dummy or exercising `closeDatabaseConnections`), leaving a stale async-context reference. Every later test then fell back to a fresh checkout for each hook, so transaction-based database cleaning could no longer isolate a test (its `beforeEach` transaction, body and `afterEach` ran on different connections), which surfaced against MS-SQL as a cascade of "A transaction is already running" failures. The per-test `ensureConnections` reuses the suite pin while it is healthy and transparently re-establishes a per-test pin when it is not.
