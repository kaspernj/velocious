Stop silently swallowing unexpected frontend-model command failures. The frontend-model controller still returns the client-safe `Request failed.` response, but now also emits genuinely unexpected errors on the `framework-error`/`all-error` event bus (with `context.frontendModelEndpoint`) so process-level reporters capture them. Expected user-flow errors stay off the bus: `ValidationError` messages are forwarded to the client (e.g. `Name can't be blank`) instead of being masked, and `error.velocious`-annotated / `safeToExpose` errors keep their own message. Frontend-model client errors now also carry server-provided `debugErrorClass`/`debugBacktrace` when the backend opts to expose them.
