## Added

- Frontend-model HTTP transport now supports a bounded, truthful deadline built on awaitery's `timeout`. `FrontendModelBase.configureTransport` accepts `timeout` (ms) and an optional caller/session `signal`; the deadline covers connection, response-header wait, and JSON/error response-body consumption. On expiry the live `fetch` — and any `websocketClient` adapter request whose `post` forwards the provided `signal` — is aborted and awaitery's `TimeoutError` is thrown, so callers can classify a timeout (`error instanceof TimeoutError` from `awaitery/build/timeout.js`) without parsing browser/network error text. A caller-supplied abort is composed with the deadline signal and stays distinguishable from a timeout, and the awaitery timer plus the composed abort listeners are cleaned up on every completion path. The composed signal is passed into `websocketClient.post(path, body, {headers, signal})`.
