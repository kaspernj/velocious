`generate:frontend-models` no longer aborts on source it can't fully parse. Its return/param-type source scanner now skips a class body whose braces it can't match (e.g. a regex literal whose quotes look like string delimiters) instead of throwing, and abstract base resources without a static `ModelClass` (an app's shared `BaseResource` that other resources extend) are treated as resource-less during generation and skipped by the realtime websocket-publisher registration instead of throwing `requires a static ModelClass`.
