Add Beacon, an opt-in cross-process broadcast bus that lets `configuration.broadcastToChannel(...)` deliver events to subscribers in any peer process. Run the broker daemon with `npx velocious beacon`, configure peers via the new `beacon: {host, port}` option (or `VELOCIOUS_BEACON_HOST` / `VELOCIOUS_BEACON_PORT` env vars), and the HTTP server / background-jobs main / background-jobs worker auto-connect during startup. `beacon: {inProcess: true}` runs Beacon in-memory via a module-level singleton — useful for tests and single-process deployments. Broker unreachable or mid-session disconnects are surfaced on `configuration.getErrorEvents()` as `framework-error` / `all-error` with `context.stage` set to `"beacon-connect"` or `"beacon-disconnect"`, with an unhandled-rejection safety net when no listener is attached. See `docs/beacon.md`.
