Add Beacon, an opt-in cross-process broadcast bus that lets `configuration.broadcastToChannel(...)` deliver events to subscribers in any peer process. Run the broker daemon with `npx velocious beacon`, configure peers via the new `beacon: {host, port}` option (or `VELOCIOUS_BEACON_HOST` / `VELOCIOUS_BEACON_PORT` env vars), and the HTTP server / background-jobs main / background-jobs worker auto-connect during startup. See `docs/beacon.md`.
