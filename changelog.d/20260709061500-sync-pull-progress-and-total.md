Report pull progress from the sync client. `sync(query, {onProgress})` and `pull({onProgress})` (`src/sync/sync-client.js`) thread an optional `onProgress` callback down to the underlying `SyncApiClient.pullChanges` per-page emit, so app code can observe `{pages, syncedCount, total}` while a scope pulls (cumulative across scopes for multi-scope pulls). Declaring a scope pulls it immediately, so a newly declared scope's initial import is reported through `sync()`, while `pull()` reports later refreshes of the already-declared scopes. To supply the `total` denominator, the change-feed endpoint (`SyncModelChangeFeedService.changes`) additively returns `total` — the scope's pending change count from the request cursor to the snapshot bound, computed with a COUNT rather than by materializing the rows (shared cursor-filter query with the page read). `pullChanges` reports a stable total across pages (already-applied rows plus the server's count from the current request cursor stays constant as the cursor advances) and emits once with `total: 0` for a pull with nothing to sync. The pull result (`SyncChangesResult`) also carries `total`. Fully additive: `sync(query)` and `pull()` without options are unchanged, older app callers are unaffected, and a server that does not report the count leaves `total` at `0` on every page. This lets a full-import-with-progress screen render "syncedCount of total" straight off the sync client. See `docs/sync-client.md` and `docs/offline-sync.md`.
