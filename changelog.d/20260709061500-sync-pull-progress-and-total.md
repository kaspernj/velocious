Report pull progress from `SyncClient.pull()`. `pull({onProgress})` (`src/sync/sync-client.js`) now threads an optional `onProgress` callback down to the underlying `SyncApiClient.pullChanges` per-page emit, so app code can observe `{pages, syncedCount, total}` while a scope pulls (cumulative across scopes for multi-scope pulls). To supply the `total` denominator, the change-feed endpoint (`SyncModelChangeFeedService.changes`) additively returns `total` — the scope's pending change count from the request cursor to the snapshot bound, computed with a COUNT rather than by materializing the rows (shared cursor-filter query with the page read). `pullChanges` reports a stable total across pages (already-applied rows plus the server's count from the current request cursor stays constant as the cursor advances) and emits once with `total: 0` for a pull with nothing to sync. The pull result (`SyncChangesResult`) also carries `total`. Fully additive: `pull()` with no arguments is unchanged, older app callers are unaffected, and older servers that omit `total` fall back to `0`. This lets a full-import-with-progress screen render "syncedCount of total" straight off `pull({onProgress})`. See `docs/sync-client.md` and `docs/offline-sync.md`.
