- Frontend model resources now default to the full CRUD ability set (`create`, `destroy`, `read`, `update`) when `static abilities` is omitted, instead of read-only. Resources that want a different set still declare it explicitly (for example `static abilities = ["read"]` for a read-only API). This removes the boilerplate `static abilities = ["create", "destroy", "read", "update"]` that every CRUD resource previously needed to repeat.
