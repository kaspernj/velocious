# Changelog

- Added `ServerSequenceAllocator` (`src/sync/server-sequence-allocator.js`): monotonically increasing server sync sequences from an auto-created AUTO_INCREMENT id table (`velocious_server_sequences` by default, custom `tableName`/`insertData` for existing bare tables), with serialized parallel allocations, lazy configuration resolution and a memory fallback, plus a `withServerSequence(ModelClass, {allocator, column})` mixin that assigns an initial sequence in `beforeCreate` and defines `advance<Column>()` through the generated typed setter. Allocations read the id from the insert statement itself (`OUTPUT INSERTED`/`RETURNING`) because MSSQL's `SCOPE_IDENTITY()` is batch-scoped, and the auto-created table survives transactional-DDL rollbacks of a surrounding failed save (readiness is only cached when the table was not created inside an active transaction; drivers gained `insideTransaction()`). Empty-data inserts now also emit `DEFAULT VALUES` on pgsql.
- `SyncResourceBase` gained declarative `static quickSearchColumns` (expands the `quickSearch` pseudo-column index search to an OR of LIKE conditions over the declared columns with client-safe rejection of invalid operators/values) and `static writableAttributes` consumed by `normalizeWritableAttributes(attributes)` — a per-attribute schema (string/date/json/raw with trim, bounds, required-blank rejection, camelCase+snake_case input keys and pinnable per-attribute messages) built on the standalone `normalizeAttributesWithSchema` (`src/sync/sync-attribute-normalizer.js`). See `docs/offline-sync.md`.
