Queue sync mutations by default for declared models: every model declaring `static sync` (`true` or a declaration object) now gets automatic create+update mutation tracking when the derived sync client starts — no `track` key needed. `static sync = {track: false}` opts a model out (for models written by non-user flows), `track: true` still adds destroys, and an operations array/`{operations}` still narrows. Tracked queueing now defers through the model connection's `afterCommit` hook, so queued syncs never reference rolled-back rows and the immediate replay attempt cannot race an uncommitted transaction. The internal echo-suppression WeakSet is generalized into a public API: `syncClient().withoutTracking(callback)` suppresses tracked queueing for the whole async duration of the callback (legacy pull paths, importers, sign-in backfills), and `markRemoteApply(record)` is the record-precise form the derived pull/realtime appliers use internally. Behavior change: apps whose `static sync` models were previously pull-only and unlisted in `track` should declare `track: false` or wrap server-originated writes in the suppression API before starting the client.
