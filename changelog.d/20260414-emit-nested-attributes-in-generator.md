- **Breaking:** convert `FrontendModelBaseResource.nestedAttributes` from a static field to an instance method `nestedAttributes(arg)`. Lets the policy be request-aware (e.g. `arg.locals.isAdmin` controls `allowDestroy`). Resources that previously declared `static nestedAttributes = {tasks: {...}}` migrate to `nestedAttributes(arg) { return {tasks: {...}} }`.
- **Breaking:** `permittedParams(arg)` is now strict by default. Submitting an attribute (nested or not) that is not in the returned permit raises an error. Default `attributes` is derived from `static attributes` minus the conventional auto-managed columns (`id`, `createdAt`, `updatedAt`); previously the default was `null` (permit all writable). Resources that need to write attributes outside `static attributes` must override `permittedParams(arg)` and return them explicitly.
- Generated frontend-model `resourceConfig()` now emits `nestedAttributes` by invoking the backend resource's `nestedAttributes()` method at generation time. The client `save()` walker reads this to decide which dirty children to ship in the payload — without it the walker has nothing to act on and nested writes silently no-op. Function-valued options (e.g. `rejectIf`) are not serialized into the generated frontend model — they remain server-only policy.
