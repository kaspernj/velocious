Add `useLiveQuery` — a reactive local-model-change query primitive. A new framework record-change bus (`src/database/record-changes.js`) emits once per commit from `save()`/`destroy()` through the connection's `afterCommit` hook, so local writes, pulled changes, and realtime pushes converge on one uniform signal; emission is skipped when nothing observes the model class (no overhead for server saves or unwatched models) and a rolled-back save emits nothing. `useLiveQuery(Model.where({...}))` (`src/database/use-live-query.js`, wrapping the React-free `LiveQuery` controller in `src/database/live-query.js`) runs the query, subscribes to its model class on the bus, and re-runs — microtask- or debounce-coalesced and protected against stale responses — whenever a watched model commits, returning `{results, loading, error}`. The sync client wraps its pull (`SyncClient.pull`) and realtime (`SyncRealtimeBridge.applyMessage`) apply loops in `recordChanges.batch(...)`, so a whole pull or push triggers one re-run per live query instead of one per applied row. Invalidation is by model class (no per-condition matcher). Additive and client-bundle safe; see `docs/live-queries.md`.
