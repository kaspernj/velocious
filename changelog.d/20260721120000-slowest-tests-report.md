Added a slowest-tests report to `velocious test`: each test's wall-clock duration (the whole per-test lifecycle, retries included) is recorded and a `Slowest N tests:` section — duration, full description and `file:line` — is printed after the run summary. Defaults to the top 10; tune with `VELOCIOUS_SLOW_TEST_COUNT` (`0` disables), and it is skipped for single-test runs. `TestRunner#getSlowestTests(limit)` exposes the same data programmatically (slowest first; `limit` of `0` returns all).
