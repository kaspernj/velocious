# Changelog

- Fix a background-jobs worker wedge that could silently freeze the whole queue. A job's worker slot is now freed as soon as the job/child completes, fully decoupled from reporting the result to the main; result reporting is durable (retried until it lands) and runs in the background, so a slow or transiently-unreachable main can neither leak worker slots (previously this drove the worker to stop accepting jobs) nor lose a terminal completion report.
- Add worker↔main liveness: workers advertise heartbeat support in the hello handshake, send periodic heartbeats, and enable TCP keepalive. The main drops a heartbeat-capable worker that goes silent past `workerStaleTimeoutMs` and releases its leases, so a wedged or half-open worker self-heals instead of stalling the queue until a human intervenes. Workers that do not advertise heartbeat support (e.g. an older release mid rolling deploy) are exempt from stale eviction and continue to be handled by socket-close disconnect recovery.
