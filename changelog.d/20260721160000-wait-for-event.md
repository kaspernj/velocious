Added `waitForEvent(emitter, eventName, {timeoutMs, filter})` (exported from `velocious/build/src/testing/test.js`) so tests can wait for a real signal — a background job finishing, a model lifecycle event, a websocket message — instead of sleeping a fixed duration. It resolves the instant a matching event fires (optionally filtered on the emitted arguments), rejects on timeout, and always removes its listener. Pairs with awaitery's `waitFor` for condition polling; see the "Avoiding fixed sleeps" testing guideline.
