- `FrontendModelBase.save()` now sends only **changed attributes** on the update path. Previously the walker sent `this.attributes()` which included framework-managed fields like `id`, `createdAt`, `updatedAt` that resources never list in `permittedParams` — under the strict-by-default permit (since 1.0.321) those rejections broke every update flow. Create payloads still send the full assigned attribute hash since the user-set hash on a freshly-built model only contains writable fields. Update behavior now mirrors what the explicit `update(attrs)` path has always done.
