Add a derived realtime bridge to the sync client: `syncClient().subscribeRealtime(context)` / `unsubscribeRealtime()` / `realtimeStatus()` / `waitForRealtimeApplied()` subscribe websocket channels and apply pushed changes through the same derived resource applier as pulls — echo suppression keeps tracked models from re-queueing applied pushes, unknown resource types fail loudly, and messages whose `echoOrigin` matches the configured `localOrigin` are dropped as own-device echoes. The app footprint is a new optional `sync.client.realtime = {createClient, channels?, localOrigin?, pullOnReconnect?}` configuration block (channels with runtime params like eventId come from the `channels(context)` callback; the framework injects `authenticationToken` into every subscription's params), plus an optional static channel declaration on models: `static sync = {realtime: {channel, params?}}`. With `pullOnReconnect` (default true), subscription readiness and post-drop resumes fire a coalesced `pull()` that closes offline gaps while low-level reconnect stays in the websocket client.
