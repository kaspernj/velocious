SQLite migrations can now add foreign-key constraints. `migration.addForeignKey(...)` and `driver.addForeignKey(...)` go through a reusable `TableRebuilder` that performs SQLite's standard "create temp table → copy rows → drop → rename" sequence with `PRAGMA foreign_keys` toggling around the swap. `CreateTable` emits table-level `CONSTRAINT <name> FOREIGN KEY` clauses so the user-supplied constraint name survives.
