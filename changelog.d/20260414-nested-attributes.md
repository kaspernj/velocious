- Add Rails-style `nestedAttributes` support for frontend-model `save()`. Resources declare writable nested relationships via `static nestedAttributes = {childName: {allowDestroy?, limit?, rejectIf?}}`. The client walker on `save()` collects dirty children (new records, records marked with `markForDestruction()`, records with changed attributes, records with dirty descendants) and ships them in a `nestedAttributes` payload. The backend unpacks the payload, authorizes each child against its own resource's abilities, and applies destroys → updates → creates in order, all inside a single DB transaction.
- `FrontendModelBase` gains `markForDestruction()` and `markedForDestruction()` for queuing destroy-on-next-parent-save.
- `FrontendModelResourceConfig` typedef updated to include `attributes`, `relationships`, and `nestedAttributes`, matching the generator output.
- Fix: `setAttribute` no longer clears every cached relationship on any attribute change. It now only invalidates a `belongsTo` cache when the attribute matches that relationship's foreign key (declared or inferred from the relationship name). This preserves preloaded children and nested-save state during parent edits.
