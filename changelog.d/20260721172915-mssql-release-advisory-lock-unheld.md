Fixed the SQL Server driver leaking a `sp_releaseapplock` error when `releaseAdvisoryLock` was called for a lock the session no longer holds. SQL Server raises error 1223 ("cannot be released because it is not currently held") instead of returning a failure code, which aborted connection check-in and close during advisory-lock cleanup. The driver now recognizes that specific error number and resolves to `false`, matching the cross-driver contract already honored by MySQL/PostgreSQL/SQLite where releasing an unheld lock is a no-op.
