Own the sync realtime channel in the framework: a canonical `velocious-sync` websocket channel is registered automatically when `sync.api` is configured, its subscribe authorization delegates to the app sync resource's existing `authorizeChanges({params, scope})`, and `SyncPublisher` broadcasts the standard sync envelope (`{echoOrigin, syncs}`) on it after every published server-side change. The derived `SyncClient` subscribes its declared pull scopes to the framework channel automatically, so the app-side realtime config shrinks to `sync.client.realtime.createClient` (plus `localOrigin`).

The sync feed's scope partition is app-declared instead of framework-named: sync models declare `static syncScopeAttributes` (for example `["eventId"]` or `["accountId"]`), the publisher persists and broadcasts those attributes generically (each reads the record's same-named attribute, else the record's own id for scope-root models, overridable via `publish: {scopeAttributes: {...}}`), and the change feed serializes them under their own names. `publish: true` opts a model in with a default attribute serializer, and typed `serialize` declarations need no casts (`SyncPublishDeclaration<TModel>`).

Deprecated but still working for 1.0.503 compatibility: the publisher `broadcasts` list, the `eventId` string/resolver publish forms (pinned to the `event_id` column and `eventId` scoping param), the `sync.client.realtime.channels` callback, the model-level `static sync = {realtime: {channel}}` declaration, and the change feed's `eventId` wire field for sync models declaring no scope attributes.
